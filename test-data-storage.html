Data Storage System

The SkillSphere application now uses persistent data storage that saves all changes to the browser's localStorage.

Key Features:

  • ✅ Persistent: Data survives browser restarts
  • ✅ Real-time: Changes are saved immediately
  • ✅ Automatic: No manual save required
  • ✅ Export/Import: Backup and restore data

Browser Console Commands

Open browser developer tools (F12) and try these commands:

Check Current Data

// View all stored data console.log(dataManager.loadData()); // View specific data const data = dataManager.loadData(); console.log('Users:', data.users); console.log('Skills:', data.skills); console.log('Projects:', data.projects);

// Add new user dataManager.addUser({ id: "EMP007", name: "Test User", password: "password123", role: "Employee", team: "S4HANA", subTeam: "Development", manager: "Alice Johnson" }); // Add new skill dataManager.addSkill({ id: "SKILL" + Date.now(), employeeId: "EMP001", skillName: "Test Skill", proficiencyLevel: "Beginner", category: "Technical", yearsExperience: 1 });

Data Management

// Export data to JSON file dataManager.exportData(); // Clear all data dataManager.clearAllData(); // Check data size console.log('Data size:', JSON.stringify(dataManager.loadData()).length, 'bytes');

Data Storage Location

Your data is stored in:

  • Location: Browser's localStorage
  • Key: skillsphere_data
  • Format: JSON object
  • Persistence: Survives browser restarts

View Stored Data

To view your stored data:

  1. Open browser developer tools (F12)
  2. Go to Application tab
  3. Expand Local Storage
  4. Click on your domain
  5. Look for skillsphere_data

Data Structure

The stored data follows this structure:

{ "users": [ { "id": "EMP001", "name": "John Doe", "password": "password123", "role": "Employee", "team": "S4HANA", "subTeam": "Development", "manager": "Alice Johnson" } ], "employees": [...], "skills": [...], "projects": [...] }

Important Notes

  • ⚠️ Browser-specific: Data is stored per browser
  • ⚠️ Local only: Data doesn't sync across devices
  • ⚠️ No backup: Data is only stored locally
  • ⚠️ Storage limit: localStorage has size limits

Best Practices

  • ✅ Export data regularly for backup
  • ✅ Use the same browser for consistency
  • ✅ Clear data if you encounter issues
  • ✅ Check console for data storage logs
